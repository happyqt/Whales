# OTP FOCUS — Редизайн мобильного приложения ОТП Банка

Решение команды WHALES в рамках чемпионата Changellenge Cup IT 2026 (секция Product Management). 

В данном репозитории представлен концепт и интерактивный web-прототип нового интерфейса мобильного приложения ОТП Банка. Главная цель проекта — снизить когнитивную нагрузку на пользователя, упростить навигацию и повысить конверсию в кросс-продажи за счет перехода к архитектуре, выстроенной вокруг пользовательского намерения (JTBD).

## Проблематика
Интерфейсы современных мобильных банков перегружены: на одном экране смешиваются переводы, балансы, промо-предложения и редкие функции. Это ухудшает пользовательский опыт, вызывает «баннерную слепоту» и снижает эффективность продаж банка.

## Концепция решения
Мы разделили функционал приложения по ключевым действиям клиента на три логических слоя:

1. **Главная (Сделать):** Минималистичный рабочий экран для ежедневных действий. Включает переводы, интерактивную карту, балансы и только один персонализированный промо-слот.
2. **Витрина (Выбрать):** Все продукты банка структурированы в 6 понятных блоков (Карты, Платежи, Накопления, Кредиты, Инвестиции, Расходы). Пользователь заходит сюда в режиме осознанного выбора.
3. **Поиск (Найти):** Быстрый интеллектуальный поиск по намерению, сокращающий путь в редких или проблемных сценариях.

## Интерактивный прототип

В репозитории содержится интерактивный web-прототип (`index.html`), демонстрирующий логику работы концепции OTP FOCUS. Прототип реализован в виде Single-Page Application (SPA) в одном файле и работает полностью на стороне клиента.

### Технологический стек
* **Разметка и стилизация:** HTML5, Tailwind CSS (подключен через CDN для быстрой сборки UI).
* **Логика:** Vanilla JavaScript (ES6+).
* **Отрисовка:** Динамическая генерация DOM-узлов на основе внутреннего состояния (State-driven rendering) без использования тяжеловесных фреймворков.

### Архитектура и структура данных
Логика прототипа опирается на структурированные mock-данные, имитирующие ответ банковского бэкенда:
* `appData`: Содержит профиль пользователя, балансы, активные продукты, историю операций и поведенческие паттерны (например, наличие свободных средств).
* `itemCatalog`: Единый реестр всех возможных действий, продуктов и сервисов в приложении. Каждая сущность имеет набор атрибутов (иконка, теги, описание, связанные действия и пути для навигации).
* `categoryDefinitions`: Конфигурация разделов «Витрины», определяющая сетку вывода элементов из каталога на экран, а также параметры empty-state состояний.
* `promoRules`: Движок персонализации. Определяет, какой именно промо-блок показать на Главной или Витрине в зависимости от профиля пользователя.

### Ключевые технические и UX-решения
* **Адаптивный мокап:** Интерфейс приложения отрисовывается внутри CSS-мокапа смартфона, который автоматически масштабируется под высоту окна браузера.
* **Умный поиск:** Реализован алгоритм нечеткого поиска (fuzzy search) с собственной системой скоринга. Алгоритм учитывает не только прямые совпадения по названиям, но и синонимы, пути навигации и приоритезирует результаты по группам.
* **Модальные окна (Bottom Sheet):** Детальная информация по продуктам и мастер-сценариям открывается во всплывающем снизу окне с плавными transition-анимациями.
* **Микроинтеракции:** * Использование 3D-трансформаций для переворота банковской карты и бесшовного доступа к реквизитам.
  * Система всплывающих toast-уведомлений для обратной связи при совершении действий.
